# Pixifact v0.1.3

First npm release for Pixifact.

## Published Packages

- `pixifact@0.1.3`
- `pixifact-cli@0.1.3`
- `create-pixifact@0.1.3`

## Install

Create a new Pixifact project:

```bash
bun create pixifact my-game
cd my-game
bun install
bun run build
```

Use Pixifact in an existing Bun project:

```bash
bun add pixifact pixi.js
bun add -d pixifact-cli
```

## Highlights

- Runtime, Scene DSL, compiler, and authoring APIs are available from `pixifact`.
- Scene validation and compilation commands are available from `pixifact-cli`.
- `create-pixifact` scaffolds a minimal Bun + Vite + Pixifact project.
- `.scene` files remain the source of truth for AI-assisted Scene authoring.

## Verification

This release was verified after publishing with:

```bash
npm view pixifact version --registry https://registry.npmjs.org/
npm view pixifact-cli version --registry https://registry.npmjs.org/
npm view create-pixifact version --registry https://registry.npmjs.org/
```

And a real npm install path:

```bash
bun create pixifact npm-smoke
cd npm-smoke
bun install
bun run build
```

The smoke project installed `pixifact@0.1.3` and `pixifact-cli@0.1.3`, compiled Scenes successfully, and completed a Vite production build.

## Notes

- `pixifact-cli` and `create-pixifact` are Bun-first tools in this release.
- The desktop editor remains distributed from the source repository, not as an npm package.
- Repository Codex skills are still installed from source; a separate npm-distributed skills package is planned later.
